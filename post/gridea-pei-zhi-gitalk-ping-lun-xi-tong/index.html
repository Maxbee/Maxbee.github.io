Gridea配置Gitalk评论系统

2020-04-27

Gitalk 是一个基于 GitHub Issue 和 Preact 开发的评论插件，可以使用GitHub账号进行登录。

Gridea官方文档中并没有对Gitalk评论进行配置对教程，搜索了一番把自己对小站支持了一番Gitalk。其实步骤很简单，不过在配置的过程中有一些我们需要注意的地方，话不多说，直接上教程。

第一步

配置完成后，会获得Client ID 和Client Secret，打开Gridea远程设置中配置，如下图：

配置完成后保存并且把网站发布即可，发布后请用部署的地址访问网站而不是本地预览，否则会出现「未找到相关的 Issues 进行评论请联系 @_____ 初始化创建」的提示。还有一个原因是因为你的部署站点的Git仓库未打开issue，在仓库中把issue打开即可。（具体设置在仓库setting中设置）
